# create the user-article matrix with 1's and 0's
def create_user_item_matrix(dataframe):
    '''
    INPUT:
    dataframe - pandas dataframe with article_id, title, user_id columns
    
    OUTPUT:
    user_item - user item matrix 
    
    Description:
    Return a matrix with user ids as rows and article ids on the columns with 1 values where a user interacted with 
    an article and a 0 otherwise
    '''
        
    unique_articles = dataframe["article_id"].unique().tolist()
    unique_users = dataframe["user_id"].unique().tolist()
    user_item_matrix = np.zeros((len(unique_users), len(unique_articles)))
    for user in unique_users:
        user_articles = dataframe.loc[dataframe["user_id"] == user]["article_id"].unique().tolist()
        for article in user_articles:
            #user_item_matrix[user-1, unique_articles.index(int(article)] = 1
            user_item_matrix[unique_users.index(user), unique_articles.index(article)] = 1
    
    df_user_item = pd.DataFrame(user_item_matrix, columns=unique_articles,index=unique_users) 
    df_user_item.index.name = "user_id"
    df_user_item.columns.name = "article_id"
    return df_user_item # return the user_item matrix

    '''
    INPUT:
    user_id - (int) a user_id
    user_item - (pandas dataframe) matrix of users by articles: 
                1's when a user has interacted with an article, 0 otherwise
    
    OUTPUT:
    similar_users - (list) an ordered list where the closest users (largest dot product users)
                    are listed first
    
    Description:
    Computes the similarity of every pair of users based on the dot product
    Returns an ordered
    
    '''
    
    # compute similarity of each user to the provided user
    similarity_user_pair = []
    user_articles = user_item_matrix.loc[user_id]
    
    similarity_user_pair = [(idx, np.dot(user_articles, comparison_user_articles)) for idx, comparison_user_articles in user_item_matrix.iterrows() if idx != user_id]

    # sort by similarity
    similarity_user_pair.sort(key=lambda user_score: user_score[1], reverse=True)
    # create list of just the ids
    
    # remove the own user's id
        
    return similarity_user_pair # return a list of the users in order from most to least similar

def user_user_recs_part2(user_id, m=10):
    '''
    INPUT:
    user_id - (int) a user id
    m - (int) the number of recommendations you want for the user
    
    OUTPUT:
    recs - (list) a list of recommendations for the user by article id
    rec_names - (list) a list of recommendations for the user by article title
    
    Description:
    Loops through the users based on closeness to the input user_id
    For each user - finds articles the user hasn't seen before and provides them as recs
    Does this until m recommendations are found
    
    Notes:
    * Choose the users that have the most total article interactions 
    before choosing those with fewer article interactions.

    * Choose articles with the articles with the most total interactions 
    before choosing those with fewer total interactions. 
   
    '''
    # Your code here
    recs = []
    df_top_sorted = get_top_sorted_users(user_id)
    
    # Gather base recommendations from similar users
    for _, user_data in df_top_sorted.iterrows():
        recs += df.loc[df["user_id"] == user_data["neighbor_id"]]["article_id"].unique().tolist()
        if len(recs) >= m:
            break
            
    # Order by article virality
    recs_with_interactions = [(rec, df.loc[df["article_id"] == rec].shape[0]) for rec in recs]
    recs_with_interactions.sort(key=lambda rec_pair: rec_pair[1], reverse=True)
    
    # only keep Id
    recs = [rec[0] for rec in recs_with_interactions]
    
    # prune to the requested size
    while len(recs) > m:
        del recs[-1]
        
    rec_names = [df.loc[df["article_id"] == rec]["title"].unique().tolist()[0] for rec in recs]
    return recs, rec_names

def get_errs(u, s, vt, lfeature_range, item_matrix):
    sum_errs = []
    for k in num_latent_feats:
        # restructure with k latent features
        s_new, u_new, vt_new = np.diag(s[:k]), u[:, :k], vt[:k, :]

        # take dot product
        user_item_est = np.around(np.dot(np.dot(u_new, s_new), vt_new))

        # compute error for each prediction to actual value
        diffs = np.subtract(item_matrix, user_item_est)

        # total errors and keep track of them
        err = np.sum(np.sum(np.abs(diffs)))
        sum_errs.append(err)
    return sum_errs

Evaluation:

Computational performance

In order to have a fair comparison basis for evaluation, we focus on the previously identified predictable users. Having a look at the results we receive a very poor performance with the brute-force method for sharing always the same overal most viewed articles for each user, the accuracy didn't even reach 50 % for any evaluated user.

A lot better results we can achieve with the method where articles are recommended, which a similar user to the evaluation candidate, also liked. Though there are candidates where not prediction had any match

The best recommendations are extracted from SVD, with an accuracy performance >> 0.9. Such a models work exceptionally well due to the base user-item matrix we are working with is complete, and doesn't contain any nan/whitespots. However this would rather be considered as a laboratory condition, as usually no user had a chance to choose of interacting with every article. In general it is a nice workaround to only notice the articles of interaction with 1 and others with 0, in order to run the base SVD, however the price is paid by spoiling the information gain (as it would be more valuable to know which articles the user is generally not aware of, next to noticing the ignored ones). But the results taken from here are not derived from the vanilla but manipulated dataset.

If we in turn would like to go with this 3 state approach, the SVD calculation from above wouldn't handle the then newly introduces nan cases, that could be overcomed with the FunkSVD derivation.

Actual performance

The result discussion above however cannot hold for how well our approach works out if it is really applied. This is not covered by this metric. It has different reasons, one of them is that the dataset we derived the results from has been made in the past, while the world naturally kept on progressing and in the meantime the individual's perferences may have progressed and changed as well.

That being said, we can take the investigations above only as an approximation, which of our options to choose from would be most promising. To measure our actual performance we would rather have to make an experiment, by separating our set of users in two groups. The one group perceives the application as it was before, while the other group perceives the application almost the same, except of we additonally would integrate our choosen algorithm. This trial then is carried out a certain time span long, and eventually the user satisfaction is compared and the better option should be kept accordingly. How this satisfaction is measured, what users are put in which group, that the only difference the users perceive is really only the algorithm and ultimatively that the users are not aware of such a separation (to avoid bias) are crucial conditions for the success. This approach is widely known as a so called A/B-testing.
